## Cross-Phase Integration

### E2E Flows

| Flow | Status | Notes |
|------|--------|-------|
| `gf pr create --title X` on GitHub | **Complete** | forge::detect → adapter::translate → runner::run(gh) fully wired |
| `gf mr create --body X` on GitLab | **Complete** | mr alias resolves to pr; body→description translation; runner::run(glab) |
| `gf browse` on public GitHub/GitLab | **Complete** | Early-intercepted before forge::detect; parse_remote_parts → build_repo_url → webbrowser::open |
| `gf <cli not found>` | **Complete** | which::which fail → GfError::CliNotFound → install hint printed |
| `gf --remote upstream pr list` | **Complete** | --remote global arg passed to forge::detect; pr list passthrough to forge CLI |
| `gf browse` on self-hosted (TOML config) | **Partial** | resolve_forge_type in browse does not call config_lookup; ForgeNotDetected for self-hosted users |

### Wiring Check

All 8 cross-phase public APIs are consumed:
- `forge::detect` ← main.rs
- `ForgeType::cli_name()` ← main.rs
- `forge::parse_remote_parts` ← browse/mod.rs
- `cmd::build_cli` ← main.rs
- `adapter::translate` ← main.rs
- `runner::run` ← main.rs
- `browse::run` ← main.rs
- `error::cli_info` ← runner.rs

**One partial connection:** `forge::config_lookup` is not called by `browse::resolve_forge_type`. Both `config_lookup` and `load_config` are `pub(crate)` — browse would need to call them directly or have forge expose a unified detection helper. Acknowledged in browse/mod.rs. Affects BROWSE-01 for self-hosted users only; all public forge flows are complete.

## Conclusion

gf v1.0 milestone is functionally complete. All 28 in-scope requirements are satisfied across all four phases. No critical blockers exist. The codebase compiles cleanly with 96 unit tests and 25 integration tests green.

The milestone carries 7 tech debt items, the most notable being the self-hosted forge limitation in `gf browse` (BROWSE-01 partial for self-hosted users). This is a known design tradeoff — `browse::resolve_forge_type` is a local inline matcher that bypasses the config file lookup used by the main forge detection path. This should be addressed before self-hosted users are in the target audience.

CORE-04 (self-hosted forge detection via CLI auth probing) remains explicitly deferred to v2, as planned.
